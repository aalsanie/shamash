Reports class names that violate defined naming conventions.

This inspection detects class names that use banned suffixes such as Manager, Helper, or Impl, as well as abbreviated class names that reduce clarity and expressiveness. Such names obscure intent, leak implementation details, or introduce unnecessary ambiguity.

The inspection helps enforce clean code and domain-driven naming practices by encouraging explicit, descriptive class names that clearly communicate purpose and responsibility. 


// Forbidden: banned suffix
public class UserManager {
}

// Forbidden: abbreviated name
public class Utl {
}

// Allowed: explicit and descriptive
public class UserAccountService {
}

This rule is part of Shamash naming enforcement and is intended to keep codebases consistent, readable, and aligned with clean architecture principles.
