Reports classes that are declared outside the configured architectural root package.

This inspection helps detect source files that live outside the expected package hierarchy, which can indicate accidental misplacement, architectural drift, or incomplete refactoring.

Keeping all production code under a single, well-defined root package improves discoverability, enables consistent layering rules, and prevents unstructured growth.

See https://plugins.jetbrains.com/docs/intellij/inspections.html#descriptions for more information.

Example of a violation: 


// Class declared outside the architectural root
package com.example.misc;

public class RandomUtility {
}
