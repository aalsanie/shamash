Reports classes and methods that appear to be unused and have no incoming references.

This inspection helps detect dead code that is not referenced anywhere in the project, excluding known entry points such as framework-managed components or application entry methods.

Unused classes and methods increase cognitive load, obscure the true structure of the system, and often remain after incomplete refactoring or abandoned features.

See https://plugins.jetbrains.com/docs/intellij/inspections.html#descriptions for more information.

Example of a violation: 


// Unused class
public class LegacyCalculator {

    public int add(int a, int b) {
        return a + b;
    }
}

This inspection supports safe cleanup by highlighting code that can be removed or reviewed, helping keep the codebase minimal, intentional, and maintainable.
